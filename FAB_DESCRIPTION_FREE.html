This is the free demo version of UnrealMCPCore. It includes the full MCP server, the Monitor Panel, and a curated set of core tools so you can evaluate the AI-to-editor workflow. For the complete 348-tool suite, get the full version.

Give AI agents full access to your Unreal Editor. UnrealMCPCore is a pure C++ plugin that runs an MCP server inside the editor: connect any AI client (Claude Desktop, Devin, Cursor, or your own) and let it read, create, and modify Blueprints, Assets, Levels, and more. No Python or Node.js runtime required.

How It Works

  1. Install the plugin: the MCP server starts automatically when the editor opens.
  2. Connect your AI client to http://localhost:8090/mcp (HTTP) or use the included stdio proxy for clients like Claude Desktop.
  3. Build: your AI agent can create Blueprints, edit materials, spawn actors, and more.

Free vs Full Version

  • Free (this version): Full MCP server, Monitor Panel, and a limited selection of core tools across Blueprint, Asset, Level, and Material categories. Enough to experience the complete AI-driven workflow.
  • Full: All 348 tools across 30 categories including Animation, Widget/UI, PCG, Niagara, Sequencer, GAS, Behavior Tree, and 20+ more. Available here.

Key Features (included in Free)

  • MCP Server: JSON-RPC 2.0 over HTTP with auto-start, session management, and multi-agent support.
  • Monitor Panel: Real-time log of all tool calls with revert, retry, and dockable layout.
  • Client Config Helper: One-click JSON snippet for Claude Desktop, Cursor, or any MCP client.
  • stdio Proxy: Python script (stdlib only) for MCP clients that require subprocess communication.
  • Blueprint Tools: Create, inspect, compile Blueprints. Add variables, functions, components. Wire graph nodes.
  • Asset Tools: List, inspect, rename, duplicate, delete, import/export assets.
  • Level Tools: Spawn actors, set transforms, query world settings.
  • Material Tools: Create materials and instances, set parameters.

Early Access (v0.1-beta)

UnrealMCPCore is currently in version 0.1-beta. With 348 tools across 30 categories, the feature scope is enormous. Every tool has been integration-tested (2134 automated tests), but edge cases in your specific project setup may still surface.

If you encounter any issues, please don't hesitate to contact me directly. The more details you can provide, the faster I can help:

  • A description of what happened and what you expected.
  • If possible, an export of the affected request from the MCP Monitor Panel (right-click the log entry, copy the details). This gives me the exact tool call and arguments to reproduce the issue.

The same goes for feature requests: whether it's missing functionality in existing tools, new tools you'd like to see, or entirely new categories. I'm actively developing this plugin and your feedback directly shapes the roadmap.
