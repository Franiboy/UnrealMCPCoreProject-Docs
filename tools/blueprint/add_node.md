# add_node

Add a node to a Blueprint graph by K2Node class name. Supports any K2Node subclass with special configuration for common node types (function calls, custom events, variable access). The Blueprint is compiled after adding the node.

## Input

```json
{
  "asset_path": "/Game/BP_MyActor",
  "node_class": "K2Node_CallFunction",
  "function_reference": "KismetSystemLibrary::PrintString",
  "position_x": 300,
  "position_y": 100
}
```

| Parameter            | Required | Description                                                                                          |
| -------------------- | -------- | ---------------------------------------------------------------------------------------------------- |
| `asset_path`         | yes      | Blueprint asset path (e.g. `/Game/Blueprints/BP_MyActor`)                                            |
| `node_class`         | yes      | K2Node class name (see **Supported Node Classes** below)                                             |
| `graph_name`         | no       | Target graph name (default: `EventGraph`). Use `get_blueprint` to discover graph names.              |
| `function_reference` | no       | For `K2Node_CallFunction`: function name or `ClassName::FunctionName` (e.g. `PrintString`, `KismetSystemLibrary::PrintString`) |
| `target_class`       | no       | For `K2Node_CallFunction`: class owning the function. Alternative to `ClassName::` prefix.           |
| `custom_event_name`  | no       | For `K2Node_CustomEvent`: name of the custom event (e.g. `OnGameStart`)                             |
| `variable_name`      | no       | For `K2Node_VariableGet` / `K2Node_VariableSet`: variable name (e.g. `Health`)                       |
| `position_x`         | no       | X position in the graph editor (default: 0)                                                          |
| `position_y`         | no       | Y position in the graph editor (default: 0)                                                          |
| `comment`            | no       | Comment text displayed on the node                                                                   |

## Supported Node Classes

| Node Class                | Extra Parameters Required     | Description                             |
| ------------------------- | ----------------------------- | --------------------------------------- |
| `K2Node_CallFunction`     | `function_reference`          | Call a Blueprint or C++ function        |
| `K2Node_CustomEvent`      | `custom_event_name`           | Create a custom event                   |
| `K2Node_VariableGet`      | `variable_name`               | Get a variable value                    |
| `K2Node_VariableSet`      | `variable_name`               | Set a variable value                    |
| `K2Node_IfThenElse`       | *(none)*                      | Branch (if/else)                        |
| `K2Node_Knot`             | *(none)*                      | Reroute node                            |
| `K2Node_Select`           | *(none)*                      | Select node                             |
| `K2Node_SwitchInteger`    | *(none)*                      | Switch on integer                       |
| `K2Node_SwitchString`     | *(none)*                      | Switch on string                        |
| `K2Node_SwitchEnum`       | *(none)*                      | Switch on enum                          |
| `K2Node_MakeArray`        | *(none)*                      | Make array                              |
| `K2Node_TemporaryVariable`| *(none)*                      | Temporary variable                      |
| Any `K2Node_*` subclass   | *(none)*                      | Generic support for all K2Node types    |

## Function Reference Resolution (K2Node_CallFunction)

The `function_reference` parameter is resolved in this order:

1. **Explicit class** — `ClassName::FunctionName` syntax (e.g. `KismetSystemLibrary::PrintString`)
2. **target_class parameter** — function looked up on the specified class
3. **Parent class hierarchy** — searches the Blueprint's parent class chain (self-context call)
4. **Library functions** — searches all loaded classes for static BlueprintCallable functions

## Output

```json
{
  "asset_path": "/Game/BP_MyActor",
  "graph_name": "EventGraph",
  "node_id": "K2Node_CallFunction_3",
  "node_class": "K2Node_CallFunction",
  "title": "Print String",
  "position": {"x": 300, "y": 100},
  "pins": [
    {"name": "execute", "direction": "input", "type": "exec"},
    {"name": "then", "direction": "output", "type": "exec"},
    {"name": "InString", "direction": "input", "type": "FString", "default_value": "Hello"}
  ]
}
```

## Notes

- The `node_id` is auto-generated by the engine (e.g. `K2Node_CallFunction_3`). Use it with `get_blueprint_node` for detailed inspection.
- For `K2Node_CallFunction`, the function must exist and be BlueprintCallable. Static library functions (like `PrintString`) are discovered automatically.
- For `K2Node_VariableGet` / `K2Node_VariableSet`, the variable should exist on the Blueprint (use `add_blueprint_variable` first).
- The Blueprint is compiled after adding the node.
- Undo removes the node from the graph.
- Use `connect_pins` (planned) to wire the new node to other nodes.
